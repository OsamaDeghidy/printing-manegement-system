# ملخص سريع عن صفحات التطبيق وأنواع المستخدمين

## أنواع المستخدمين (User Roles)

بناءً على ملف `lib/types.ts`، يوجد الأنواع التالية من المستخدمين:

1. **consumer** - مستخدم عادي (طالب خدمة)
2. **requester** - مقدم طلب
3. **dept_employee** - موظف قسم
4. **dept_manager** - مدير قسم
5. **print_manager** - مدير المطبعة
6. **admin** - مدير النظام
7. **approver** - معتمد
8. **training_supervisor** - مشرف تدريب
9. **inventory** - مسؤول مخزون
10. **staff** - موظف

---

## الصفحات العامة (Public Pages)

### 1. الصفحة الرئيسية (`/`)
- **المحتوى**: إعادة توجيه تلقائي إلى صفحة تسجيل الدخول
- **المستخدمون**: جميع الزوار

### 2. صفحة تسجيل الدخول (`/login`)
- **المحتوى**: 
  - نموذج تسجيل الدخول بالبريد الجامعي وكلمة المرور
  - خيار "تذكرني"
  - رابط استعادة كلمة المرور
  - معلومات المستخدم التجريبي
- **المستخدمون**: جميع الزوار غير المسجلين

---

## صفحات لوحة التحكم (Dashboard Pages)

### 3. لوحة التحكم الرئيسية (`/dashboard`)
- **المحتوى**:
  - إحصائيات سريعة (الطلبات الحالية، طلبات قيد الاعتماد، التوفير الشهري)
  - قائمة الخدمات المتاحة
  - أحدث الطلبات
- **المستخدمون**: جميع المستخدمين المسجلين

### 4. صفحة الطلبات (`/orders`)
- **المحتوى**:
  - عرض جميع الطلبات في شكل بطاقات
  - معلومات كل طلب (رقم الطلب، الخدمة، مقدم الطلب، الحالة، الأولوية)
  - زر لإنشاء طلب جديد
  - أزرار لعرض التفاصيل أو إعادة الطلب
- **المستخدمون**: جميع المستخدمين المسجلين

### 5. صفحة تفاصيل الطلب (`/orders/[orderId]`)
- **المحتوى**:
  - تفاصيل كاملة للطلب (الحقول والقيم)
  - المرفقات (ملفات وروابط)
  - سجل الحالة الزمني
  - أزرار لتحديث الحالة، تحميل الإيصال، إعادة تقديم طلب مشابه
- **المستخدمون**: جميع المستخدمين المسجلين

### 6. صفحة الخدمات (`/services`)
- **المحتوى**:
  - عرض جميع الخدمات مصنفة حسب الفئة:
    - المستندات الرسمية
    - التصميم والإبداع
    - الترويج والفعاليات
    - الخدمات الطبية
  - معلومات كل خدمة (الاسم، الوصف، هل تتطلب اعتماد)
- **المستخدمون**: جميع المستخدمين المسجلين

### 7. صفحة طلب خدمة (`/services/[serviceSlug]`)
- **المحتوى**:
  - معلومات الخدمة (الاسم، الوصف، زمن الإنجاز المتوقع)
  - نموذج تقديم الطلب مع الحقول المخصصة
  - معلومات المرفقات المطلوبة
- **المستخدمون**: جميع المستخدمين المسجلين

### 8. صفحة الزيارات (`/visits`)
- **المحتوى**:
  - قائمة بطلبات الزيارات
  - نموذج لطلب زيارة جديدة
  - معلومات كل زيارة (النوع، التاريخ، الوقت، الحالة)
- **المستخدمون**: جميع المستخدمين المسجلين

### 9. صفحة الإشعارات (`/notifications`)
- **المحتوى**: (غير مكتملة - تحتاج مراجعة)
- **المستخدمون**: جميع المستخدمين المسجلين

### 10. صفحة الإعدادات (`/settings`)
- **المحتوى**:
  - البيانات الشخصية (الاسم، البريد، القسم، رقم التواصل)
  - تفضيلات الإشعارات (البريد، داخل التطبيق، تحديثات الطلبات، إلخ)
  - تغيير كلمة المرور
- **المستخدمون**: جميع المستخدمين المسجلين

## صفحات الإدارة (Admin Pages)

جميع صفحات الإدارة محمية بـ `ProtectedRoute` وتتطلب أدوار محددة.

### 11. نظرة عامة للمدير (`/admin/overview`)
- **المحتوى**:
  - مؤشرات الأداء الرئيسية (KPI)
  - الطلبات النشطة
  - طلبات بانتظار الاعتماد
  - تنبيهات المخزون
  - نسبة التوفير
- **المستخدمون**: `print_manager`, `admin`

### 12. إدارة المستخدمين (`/admin/users`)
- **المحتوى**:
  - قائمة جميع المستخدمين
  - إضافة مستخدم جديد
  - تعديل بيانات المستخدم
  - تفعيل/إيقاف المستخدمين
  - عرض معلومات المستخدم (الاسم، البريد، القسم، الدور، الحالة)
- **المستخدمون**: `admin` فقط

### 13. إدارة الجهات (`/admin/entities`)
- **المحتوى**: (غير مكتملة - تحتاج مراجعة)
- **المستخدمون**: `admin`, `print_manager`

### 14. إدارة الخدمات (`/admin/services`)
- **المحتوى**: (غير مكتملة - تحتاج مراجعة)
- **المستخدمون**: `admin`, `print_manager`

### 15. إعدادات الاعتماد (`/admin/approvals`)
- **المحتوى**:
  - خيارات نظام الاعتماد:
    - تفعيل شامل لجميع الخدمات
    - تفعيل انتقائي
    - إيقاف نظام الاعتماد
  - قائمة الخدمات مع حالة الاعتماد لكل خدمة
  - تعديل المعتمدين لكل خدمة
- **المستخدمون**: `admin`, `approver`

### 16. إدارة المخزون (`/admin/inventory`)
- **المحتوى**: (غير مكتملة - تحتاج مراجعة)
- **المستخدمون**: `admin`, `print_manager`, `inventory`

### 17. إدارة الأسعار (`/admin/pricing`)
- **المحتوى**: (غير مكتملة - تحتاج مراجعة)
- **المستخدمون**: `admin`, `print_manager`

### 18. التقارير (`/admin/reports`)
- **المحتوى**: (غير مكتملة - تحتاج مراجعة)
- **المستخدمون**: `admin`, `print_manager`, `dept_manager`

### 19. السجلات (`/admin/logs`)
- **المحتوى**: (غير مكتملة - تحتاج مراجعة)
- **المستخدمون**: `admin`, `print_manager`

### 20. إعدادات النظام (`/admin/settings`)
- **المحتوى**: (غير مكتملة - تحتاج مراجعة)
- **المستخدمون**: `admin`, `print_manager`

## ملاحظات مهمة

1. **نظام الحماية**: جميع الصفحات الإدارية محمية بـ `ProtectedRoute` الذي يتحقق من:
   - تسجيل الدخول
   - الدور المطلوب

2. **التوجيه الديناميكي**: الصفحات الديناميكية (`[orderId]`, `[serviceSlug]`) تستخدم `dynamic = 'force-dynamic'`

3. **البيانات**: التطبيق يستخدم بيانات تجريبية (mock data) مع إمكانية الاتصال بـ API

4. **التصميم**: التطبيق يستخدم Tailwind CSS 4 مع دعم RTL (من اليمين لليسار) للغة العربية
